# Quill v1.1.4

**The footer now shows what Claude actually used — not a placeholder.**

A focused follow-up to the model and effort controls from 1.1.3. When you leave Quill on its own judgment, Claude Code picks a model and an effort level for you — and now the footer tells you what it picked, instead of just repeating your setting back at you.

## Changes

- **The inherit option is now called AUTO.** The model and effort menus previously labeled their "let Claude decide" choice **DEFAULT**; it now reads **AUTO**, which is what it actually does — leave the flag off and let Claude Code choose. Nothing about the behavior changed, only the name.
- **The footer reports the real answer.** After a request, the footer chip shows the **actual model** Claude Code ran — and, new in this release, the **effective effort** it ran at — both read straight from the live run rather than assumed from your menu. So when you're on AUTO, you can finally see which model and how much effort a reply was made with.
- **A clearer tooltip.** Hovering the chip now distinguishes an auto-selected run ("Model: Auto — last observed OPUS") from a choice you made yourself ("Model: OPUS (chosen for the next request)"), so it's never ambiguous whether you're seeing your pick or Claude's.

Nothing changes on disk — documents and sidecars remain fully compatible with every previous release in both directions.

## Install

Inside Truss, Quill is shared as a ready-to-run app on the team's Google Drive — copy `Quill.app` from the shared drive (in Finder, from your synced folder) to your Applications folder and open it. The [README](../../README.md#installing-quill) has the details and the [User Guide](../USER_GUIDE.md) a full walkthrough. The Claude features require the [Claude Code](https://claude.com/claude-code) CLI on the same Mac.

## Known limitations

- macOS only. Not yet Apple-notarized — a self-built or website-downloaded copy trips Gatekeeper on first launch (open from the synced Drive folder, or use **System Settings → Privacy & Security → Open Anyway**).
- Single-user: multiple documents open as tabs in one window; no multi-user collaboration or cloud sync.
