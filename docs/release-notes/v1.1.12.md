# Quill v1.1.12

**Editing across a line break now tracks cleanly.**

- **Hard breaks are tracked like any other text.** When you edit across a soft line break (Shift+Enter — common in addresses, signatures, and poetry) in Suggesting mode, the change is now a proper tracked suggestion: Accept removes the break along with the rest of the edit, and Reject restores it exactly. Previously the break could survive an accepted deletion, leaving a stray line break the edit never intended.

This closes a real data-corruption path — a human editing across a break and accepting could get a document that wasn't the edit they made.

No file-format or data changes. Documents, comments, suggestions, chat threads, and linked sessions carry over unchanged.

## Install

Inside Truss, Quill is shared as a ready-to-run app on the team's Google Drive — copy `Quill.app` from the shared drive (in Finder, from your synced folder) to your Applications folder and open it. The [README](../../README.md#installing-quill) has the details and the [User Guide](../USER_GUIDE.md) a full walkthrough. The Claude features require the [Claude Code](https://claude.com/claude-code) CLI on the same Mac.

## Known limitations

- macOS only. Not yet Apple-notarized — a copy downloaded through a browser (e.g. the Drive website) is typically quarantined and trips Gatekeeper on first launch; open the Finder-synced copy instead, or use **System Settings → Privacy & Security → Open Anyway**.
- Single-user: multiple documents open as tabs in one window; no multi-user collaboration or cloud sync.
- Suggesting mode cannot yet merge, split, or restructure paragraphs, list items, or other blocks as a tracked change. Editing across a hard line break is now supported; asking Claude to _create_ or _remove_ a line break (join or split lines) is still in progress.
