# v1.1.15

**Fixes silent data loss.** Update if you keep links between your documents.

## Relative links are no longer deleted when Quill saves

Opening a Markdown file that links to another file next to it — `[User Guide](docs/USER_GUIDE.md)`,
`[the plan](../planning/q3.md)` — and letting Quill save could **delete the link's destination**.
The words stayed; the link they pointed to was gone from the file on disk, with no warning.

It affected only links written as a plain relative path. Links to a website (`https://…`), links
starting with `./` or `/`, and same-page anchors (`#section`) were never affected. Quill's own
README lost three links this way, which is how the bug was found.

Underneath, Quill's editor was rejecting a path containing a `/` as an invalid link address,
dropping the link when the file was loaded and then writing the file back without it.

Two things were fixed, because either one alone would have left the bug in place:

- Loading and saving a document now preserves relative links exactly as written.
- Editing a link no longer rewrites it. Previously, opening a relative link in the link editor and
  pressing **Apply** — even after changing only its label — turned `docs/GUIDE.md` into
  `https://docs/GUIDE.md`. The same happened to relative links you typed or pasted, and to link
  edits proposed by Claude. The **Open** button also no longer offers to open a relative document
  link in your browser, which never worked.

Typing a bare domain like `example.com` in the link editor still becomes `https://example.com`, as
before.

Links using `javascript:`, `data:`, and similar are still refused, including versions that try to
hide the scheme behind whitespace.

## Open a document in Quill from Claude Code

`/open-in-quill <file.md>` in a Claude Code session working in the Quill repo opens that file in
Quill, already linked to the session you are talking to — so Claude answers with that
conversation's memory instead of asking you to pick a session.

The first time you open a particular document this way, Quill asks you to approve the link once.
That is deliberate: the linked session travels in the document's `.comments.json` companion file,
and Quill will not trust a session named by a file it has not been told to trust. After you
approve it once, later opens of that document are silent.

This replaces the Claude Code plugin removed in v1.1.7 and needs no plugin or marketplace install.

## Known limitations

Still not preserved when a document is saved, unchanged from previous versions: empty link targets
(`[text]()`), Windows drive paths, and custom schemes such as `obsidian://` or `ipfs://`. These are
dropped silently. If you rely on any of them, keep a backup outside Quill.
