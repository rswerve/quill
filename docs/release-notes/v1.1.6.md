# Quill v1.1.6

**One label on the model chip, not two.**

A small refinement following 1.1.4 and 1.1.5. On AUTO, the footer chip had been showing both the observed value and an "AUTO" label side by side — e.g. `OPUS · AUTO`, which said the same thing twice. Now it just shows the observed value on its own (`OPUS`). The "AUTO" label appears only before anything has been observed — that is, until Claude Code has actually run and reported a model or effort — and steps aside once there's a real answer to show.

Nothing changes on disk — documents and sidecars remain fully compatible with every previous release in both directions.

## Install

Inside Truss, Quill is shared as a ready-to-run app on the team's Google Drive — copy `Quill.app` from the shared drive (in Finder, from your synced folder) to your Applications folder and open it. The [README](../../README.md#installing-quill) has the details and the [User Guide](../USER_GUIDE.md) a full walkthrough. The Claude features require the [Claude Code](https://claude.com/claude-code) CLI on the same Mac.

## Known limitations

- macOS only. Not yet Apple-notarized — a self-built or website-downloaded copy trips Gatekeeper on first launch (open from the synced Drive folder, or use **System Settings → Privacy & Security → Open Anyway**).
- Single-user: multiple documents open as tabs in one window; no multi-user collaboration or cloud sync.
