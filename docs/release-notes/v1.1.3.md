# Quill v1.1.3

**Work across many documents, chat with Claude about the whole thing, and never lose an edit.**

This is a big one — six days of work gathered into a single release. Quill grew from a one-document window into a proper multi-document workspace, learned to hold a whole-document conversation with Claude, started tracking formatting changes the way it already tracks text, and quietly took over the job of saving your work. The writing surface got a redesign to match. Nothing about your files changed: documents are still plain `.md`, and the sidecar just gained a few fields (more on that below).

## Highlights

- **Multiple documents, as tabs.** Open as many documents as you like in one window. A tab strip sits below the top bar; each tab is fully independent — its own editor, comments, suggestions, session link, zoom, and mode — and background tabs stay live, so switching between them is instant. **New (Cmd+N)** and the **+** button add an Untitled tab; **Open… (Cmd+O)** focuses a document that's already open instead of opening it twice. Closing a tab with unsaved changes still asks first, and closing the last tab leaves a fresh Untitled one rather than an empty window.
- **Your session comes back.** The set of open tabs, their order, and the active tab are remembered between launches, browser-style — reopen Quill and you're back where you left off. And if the app or your machine goes down mid-edit, the next launch offers to **Recover** the documents that had unsaved changes (or **Discard** and start clean). A recovered saved document is usually already up to date on disk, because it autosaves as you go.
- **Document chat.** Alongside `@claude` replies inside a comment thread, there's now a **Chat** panel for talking about the whole document — open it from the review panel's **Comments / Chat** toggle, with **Cmd+/**, or from the empty-document hint. Ask for a pass over the draft and Claude's edits come back as ordinary tracked suggestions you Accept or Reject. Chat and its suggestions are linked both ways: a **"↳ from chat"** chip on a suggestion jumps to the message that produced it, and a **"→ N suggestions"** chip on the message jumps to the edits. The conversation is saved with the document.
- **Tracked formatting.** In Suggesting mode, formatting changes are now tracked just like text changes. Making text **bold**, _italic_, ~~struck through~~, or `inline code` applies the style right away and drops a **Formatting** card in the margin — Accept keeps the style, Reject reverts exactly what changed. One gesture is one suggestion, even across a broken-up selection. Two related changes here: **links are no longer edited silently in Suggesting mode** (Quill shows a "switch to Editing to change links" notice instead of committing an untracked change), and **underline was removed** entirely, since Markdown can't preserve it.
- **A redesigned writing surface — the Studio.** The formatting controls now live in a vertical **rail down the left edge** rather than a top toolbar. The review panel gained the **Comments / Chat** toggle and an independent, document-ordered **history list** of comment threads. And the whole app ships with **bundled offline fonts** — Source Serif 4 for the document, Instrument Sans for the interface, JetBrains Mono for status and metadata — so your typography looks right with no network and no font CDN.
- **Pick Claude's model and effort.** The footer gained two small menus that let you choose which Claude **model** and **effort** level the next `@claude` request or chat turn should use, or leave them for Claude Code to decide. Your choices persist across launches. (This is a first pass; it gets more honest in the next release.)
- **Autosave.** A document with a saved path now saves itself in the background — about **2 seconds after you stop typing** (and no later than 15 seconds if you keep going), and immediately whenever focus leaves it: switching tabs, the window losing focus, or quitting. Untitled documents have nowhere to save to, so they stay manual until your first Save. A small footer indicator shows _Saving…_ / _Saved_ so you always know where you stand, and autosave never interrupts you with a dialog — only a manual Save reports a problem loudly.
- **Conflict-safe saving.** Every save — by hand or automatic — now checks whether the file or its sidecar changed on disk since you opened it. If something else touched it, the save pauses and a banner offers **Overwrite**, **Save a Copy**, or **Reload**, so neither your version nor the other one is lost without you deciding. Under the hood, saves are atomic (written to a temp file and renamed into place), and a save that wouldn't change a byte is skipped entirely.

## Also in this release

- An empty document no longer shows a phantom scrollbar.
- The footer's word and character counts are now **selection-aware** — select some text and the counts report the selection.
- Clearer wording when a proposed `@claude` edit can't be located in the document and has to be skipped.

Documents are still plain `.md`. The sidecar moves to **version 2** — it now carries the document chat thread alongside comments, suggestions, and the linked session — but older and newer releases still read each other's files in both directions, so nothing you've already written is stranded.

## Install

Inside Truss, Quill is shared as a ready-to-run app on the team's Google Drive — copy `Quill.app` from the shared drive (in Finder, from your synced folder) to your Applications folder and open it. The [README](../../README.md#installing-quill) has the details and the [User Guide](../USER_GUIDE.md) a full walkthrough. The `@claude` features require the [Claude Code](https://claude.com/claude-code) CLI on the same Mac.

## Known limitations

- macOS only. Not yet Apple-notarized — a self-built or website-downloaded copy trips Gatekeeper on first launch (open from the synced Drive folder, or use **System Settings → Privacy & Security → Open Anyway**).
- Single-user: multiple documents open as tabs in one window; no multi-user collaboration or cloud sync.
