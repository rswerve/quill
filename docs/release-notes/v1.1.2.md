# Quill v1.1.2

**A round of fixes from real use, plus a couple of quality-of-life touches.**

This patch is all polish on top of 1.1.0/1.1.1 — nothing changes on disk, and documents and sidecars remain fully compatible with every previous release in both directions. The theme is making the `@claude` integration and the reading surface behave the way you'd expect.

## Fixes

- **`@claude` finds the Claude CLI reliably.** Spawning a reply now resolves the `claude` binary the same way your shell would — including a configured/installed path and a login shell's `PATH` — instead of assuming one fixed location, so `@claude` works even when Claude Code lives somewhere non-standard.
- **Closing, deep-linking, and the desktop menu are steadier.** Hardened the window-close capability, brought the app window to the front when opened via a `quill://` deep link, and fixed a stale-closure bug in the native menu so menu actions always act on the current document.
- **Retry a failed `@claude` reply in place.** If a reply errors out, you can retry it directly from the comment without retyping — no need to delete and re-add the `@claude` mention.
- **Re-run a cancelled `@claude` call.** Cancelling a reply now leaves a **Re-run** button on the comment so you can kick the call off again, rather than being stuck with a dead reply. (The rarely-working "Start new session" button was removed in the same pass.)
- **Comment placeholders hint at `@claude`.** The comment and reply composers now say _"@claude to get an AI response"_ so the AI trigger is discoverable instead of hidden.
- **Wider, calmer reading surface.** Reduced the page's horizontal padding to open the measure to ~85 characters, and removed a leftover page-break background that could draw faint lines through text.

## Install

| Platform              | Asset           |
| --------------------- | --------------- |
| macOS (Apple Silicon) | `…_aarch64.dmg` |
| macOS (Intel)         | `…_x64.dmg`     |

Builds are **unsigned**. On first launch, double-click **Quill**, click **Done**, then go to **System Settings → Privacy & Security → Open Anyway** (or run `xattr -dr com.apple.quarantine /Applications/Quill.app`). The old right-click → Open shortcut was removed in macOS Sequoia.

The `@claude` features require the [Claude Code](https://claude.com/claude-code) CLI on the same machine. See the [User Guide](https://github.com/sam-powers/quill/blob/main/docs/USER_GUIDE.md) for a walkthrough.

## Known limitations

- macOS only for now. The `@claude` integration finds the Claude CLI via Unix paths; rather than ship a partial experience, Windows and Linux installers are deferred (both platforms can [build from source](https://github.com/sam-powers/quill#building-from-source)).
- No code signing yet (Gatekeeper warning above).
- Single document per window; no multi-user collaboration or cloud sync.
