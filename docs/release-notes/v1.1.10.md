# Quill v1.1.10

**Honest edits, hardened further.**

This release deepens the work from 1.1.9: when Claude proposes a change the tracked-suggestion engine can't represent safely, Quill now refuses it plainly instead of applying something subtly wrong. It also lands a large edge-case test net so these guarantees stay true.

- **Edits that span a line break, a code block, or another author's pending suggestion now refuse honestly** — with a clear reason — rather than corrupting the document or silently doing nothing.
- **Overlapping or duplicate edits in one reply** are handled cleanly: overlaps are declined as a group, exact duplicates apply once.
- **Link edits** now match on both the visible text and the destination, so a rename targeting one of two same-named links lands on the right one.

No file-format or data changes. Documents, comments, suggestions, chat threads, and linked sessions carry over unchanged.

## Install

Inside Truss, Quill is shared as a ready-to-run app on the team's Google Drive — copy `Quill.app` from the shared drive (in Finder, from your synced folder) to your Applications folder and open it. The [README](../../README.md#installing-quill) has the details and the [User Guide](../USER_GUIDE.md) a full walkthrough. The Claude features require the [Claude Code](https://claude.com/claude-code) CLI on the same Mac.

## Known limitations

- macOS only. Not yet Apple-notarized — a copy downloaded through a browser (e.g. the Drive website) is typically quarantined and trips Gatekeeper on first launch; open the Finder-synced copy instead, or use **System Settings → Privacy & Security → Open Anyway**.
- Single-user: multiple documents open as tabs in one window; no multi-user collaboration or cloud sync.
- Suggesting mode cannot yet merge, split, or restructure blocks as a tracked change — Quill will tell you when a request needs Editing mode. Editing across a hard line break is also not yet supported as a tracked suggestion (a fix is in progress).
