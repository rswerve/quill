# Quill v1.1.14

**A stray space can't strand your comments anymore.**

Markdown collapses runs of whitespace — two spaces render as one, and a reopened file drops the extra. Quill anchored comments and tracked changes by exact position, so any collapsing space _upstream_ of an annotation used to shift it: a comment landed a character off, or a suggestion quarantined itself as "needs review" on reopen. This release makes review anchors survive the round-trip, so a single space never breaks the app.

- **Comments and suggestions stay put across save and reopen.** Anchors are now captured against the exact document a reopen rebuilds, so nothing downstream of a collapsing space drifts or gets set aside.
- **A comment across a double space just saves.** Highlighting text that spans a run of spaces used to block the save; now the highlight simply tucks onto the single surviving space and the save goes through — no dialog.
- **What you see is what's written.** On save, a run of spaces is normalized to the one space the editor already shows (and Markdown always meant), so the file on disk matches the page.
- **Crash recovery is lossless.** Recovered unsaved work is restored byte-for-byte — comments, tracked changes, and formatting exactly as they were — instead of being re-parsed and possibly nudged.

## Data changes

Backward-compatible and additive. New saves stamp the review sidecar with an anchor fingerprint; older files without it are treated as "needs re-anchoring." The **first time you open an existing document**, Quill re-anchors its comments and suggestions to the text — anything it can't place confidently is set aside in the review panel rather than pinned to the wrong spot, with a one-time notice. Documents, comments, suggestions, chat threads, and linked sessions otherwise carry over unchanged.

## Install

Inside Truss, Quill is shared as a ready-to-run app on the team's Google Drive — copy `Quill.app` from the shared drive (in Finder, from your synced folder) to your Applications folder and open it. The [README](../../README.md#installing-quill) has the details and the [User Guide](../USER_GUIDE.md) a full walkthrough. The Claude features require the [Claude Code](https://claude.com/claude-code) CLI on the same Mac.

## Known limitations

- macOS only. Not yet Apple-notarized — a copy downloaded through a browser (e.g. the Drive website) is typically quarantined and trips Gatekeeper on first launch; open the Finder-synced copy instead, or use **System Settings → Privacy & Security → Open Anyway**.
- Single-user: multiple documents open as tabs in one window; no multi-user collaboration or cloud sync.
- Suggesting mode cannot yet merge, split, or restructure blocks as a tracked change — Quill will tell you when a request needs Editing mode.
