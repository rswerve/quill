# Quill v1.1.7

**Housekeeping for the Truss build.**

Internal cleanup, no new features. Quill now carries the Truss application identity, the in-app "check GitHub for a newer release" banner is gone (builds are shared over the team's Google Drive, not GitHub releases), and the experimental Claude Code plugin was removed.

**One-time settings reset.** Because the application identity changed, this build starts with fresh local settings — theme, zoom, recent files, model/effort choices, and any saved permission grants reset once, and you may be asked to re-link a Claude session or reference folder the first time. **Your documents and their `.comments.json` sidecars are untouched** — those live next to your files, not in app settings.

## Install

Inside Truss, Quill is shared as a ready-to-run app on the team's Google Drive — copy `Quill.app` from the shared drive (in Finder, from your synced folder) to your Applications folder and open it. The [README](../../README.md#installing-quill) has the details and the [User Guide](../USER_GUIDE.md) a full walkthrough. The Claude features require the [Claude Code](https://claude.com/claude-code) CLI on the same Mac.

## Known limitations

- macOS only. Not yet Apple-notarized — a self-built or website-downloaded copy trips Gatekeeper on first launch (open from the synced Drive folder, or use **System Settings → Privacy & Security → Open Anyway**).
- Single-user: multiple documents open as tabs in one window; no multi-user collaboration or cloud sync.
