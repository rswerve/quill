# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

Quill is a Tauri 2 + React 19 desktop app — a multi-document Markdown editor with track-changes, inline comments, and AI collaboration from a linked Claude Code session (inline `@claude` replies in a comment thread and a whole-document chat), similar to Google Docs suggesting mode. The frontend is React/TypeScript built with Vite; the backend is a thin Rust/Tauri layer that exposes file I/O, native dialogs, and Claude-process integration to the frontend.

## Commands

```bash
npm run dev            # Frontend dev server only (no Tauri window, no file I/O)
npm run tauri dev      # Full desktop app (Tauri + frontend, hot-reload)
npm run tauri build    # Production bundle

npm run typecheck      # tsc --noEmit
npm run lint           # eslint
npm run format:check   # prettier --check
npm test               # vitest (unit + component)
npx playwright test    # end-to-end specs

cd src-tauri && cargo test && cargo clippy -- -D warnings && cargo fmt --check
```

CI (`.github/workflows/ci.yml`) runs the frontend checks (typecheck, eslint, prettier, vitest) and the Rust checks (fmt, clippy, test) on every push and PR to `main`. Match that bar before pushing.

Releases: pushing a `v*` tag triggers `.github/workflows/release.yml` (tauri-action), which builds macOS installers (aarch64 + x86_64) and attaches them to a **draft** GitHub Release. Releases are macOS-only by decision: `@claude` binary/session discovery is Unix-path-based, and we don't ship builds that can't deliver the full experience. A maintainer fills in the notes (drafts live in `docs/release-notes/`) and publishes manually. Tags are pushed by the maintainer, not by automation. Keep `version` in `package.json`, `src-tauri/Cargo.toml`, and `src-tauri/tauri.conf.json` in sync when bumping.

## Contributing

`main` is the default branch and is **not** committed to directly. To land a change:

1. Branch off `main` (e.g. `git checkout -b fix/short-description`).
2. Make the change and keep CI green locally: run the full check bar above (typecheck, lint, `format:check`, vitest, and `cargo fmt --check && cargo clippy -- -D warnings && cargo test`). Fix formatting with `npm run format` / `cargo fmt`.
3. Commit, push the branch, and open a PR against `main` with `gh pr create`. Let CI pass on the PR before merging.

Keep a PR scoped to one coherent theme. Update `PRD.md` and this file when behavior or architecture changes.

## Architecture

### Frontend (`src/`)

`App.tsx` is a **multi-tab shell**; per-document state lives in **`components/DocumentTab.tsx`** (one instance per open tab, all mounted at once, inactive ones `hidden`). No Redux or context — the shell and the active tab communicate through an imperative handle (shell → tab) and snapshot callbacks (tab → shell). Key relationships:

- **`App.tsx`** — The shell. Owns the `tabs` array, the active tab, a single active-tab chrome snapshot that feeds the left Rail / Topbar / Footer, the global keyboard shortcuts (Cmd+N/O/S, Cmd+Shift+S, Cmd+F, Cmd+/, Cmd±/0), the native-menu wiring, the cross-tab one-session-per-document claim registry, workspace persistence/recovery, and modal/guard state. It routes chrome actions to the active tab's `DocumentTabHandle`. Tab lifecycle: New / `+` / Cmd+N add an Untitled tab; Open focuses an already-open document or adds a tab; closing a dirty tab prompts Save / Don't Save / Cancel; closing the last tab leaves an Untitled one. The dirty-guard fires only on **Quit and window close**: both paths first `flushAllPendingSaves` — a bounded loop that flushes every tab (over the authoritative `tabsRef`, retaining any dirty tab whose handle is missing or whose flush threw) and re-checks each tab's `getPersistenceSnapshot` revision, repeating until the tab set is quiescent (no tab edited mid-round), then guards only the tabs **still** dirty from that stable read — Untitled tabs plus any saved tab whose flush ended failed/blocked/conflicted. Writes precede exit; a flush exception, a missing handle, or bound-exhaustion all **fail closed** (guard the tab / keep the window open). Closing a single dirty tab likewise flushes-then-guards. New and Open add tabs rather than replacing the open document.
- **`components/DocumentTab.tsx`** — All per-document state and logic for one tab: `useFileManager`, `useSaveCoordinator`, `useAutosave`, `useComments`, `useSuggestions`, `useClaudeReply`, `useDocumentChat`, the editor, the linked session, reference folder, zoom, and mode. Every save route funnels through the coordinator; autosave is wired via the single `markDirty` choke point (which also bumps the change-revision) and flushed on tab-deactivate / window-blur; it exposes `flushPendingSave` (flush + drain, returning post-flush dirty) and `getPersistenceSnapshot` (`{revision, dirty}` for the quit guard's stability loop) for the shell. Every AI-reply lifecycle outcome (finish/fail/cancel/retry) and every document-chat terminal (`useDocumentChat`'s `onTerminal`) marks dirty; terminal outcomes also flush via a POST-COMMIT tick effect, so a finished/errored/cancelled turn reaches disk immediately. An external `conflict` raises the per-tab `ConflictBanner` (sticky until Overwrite / Save a Copy / Reload); `blocked`/`failed` are presented only for a **manual** save (`presentManualSaveFailure`), while autosave surfaces them as footer status (active tab) plus a **latched** `autosaveAttention` on the tab strip (a background tab's failed/blocked flush stays visibly flagged). Exposes a `DocumentTabHandle` (save / saveAs / open / newDocument / exportPdf / setMode / setZoom / acceptAll / rejectAll / focusFind / openChat / session controls / `getWorkspaceSnapshot` / `flushPendingSave` / `getPersistenceSnapshot`) to the shell and pushes chrome + meta snapshots up (chrome carries the live `autosaveStatus`; meta carries the `conflict` flag and `autosaveAttention`). Comments are projected from their live marks on every editor update by `utils/commentReconciler.ts`: surviving unresolved marks refresh the record's range and quote, a missing unresolved mark drops the record after manual full-anchor deletion, and resolved comments are preserved because resolve deliberately strips their mark. Before Accept/Accept All, `utils/trackedCommentResolution.ts` snapshots the union of comments protected by deletion geometry and origin comments named by the accepted changes' `originCommentId`; it queues those records as resolved and strips surviving origin marks before dispatch, so accepting a comment's edit resolves that thread even when the edit lands elsewhere. Reject/Reject All keeps the protective geometry rule but has no provenance behavior. All paths preserve the pre-resolution quote/range before mark-removal reconciliation. (Link editing lives in `Toolbar.tsx`'s `LinkButton`: clicking an existing link, Cmd+K with a selection, or the rail button opens one body-portaled editor for display text + URL, with Apply/Remove/Open; shared primitives in `utils/linkEditing.ts` normalize safe hrefs, replace display text while retaining the link mark, and unlink without deleting text. In Suggesting mode link changes are **blocked** with a notice — switch to Editing to change a link — while bold/italic/strike/**code** changes are tracked as formatting suggestions; see `TrackChanges.ts` and `extensions/trackChangesPolicy.ts`.)
- **`Editor.tsx`** — Wraps Tiptap. Exposes an imperative handle (`getMarkdown()`, `setContent()`) via `forwardRef`. Do not reach past this component to manipulate Tiptap directly. The extension set includes tables, task lists, and images so those constructs round-trip through save — mirror any extension change in `src/test/utils/markdownRoundTrip.test.ts`, which asserts fidelity. StarterKit bundles Link, Underline, and Code in Tiptap v3: configure Link via `StarterKit.configure({ link: … })`, never as a separate extension (duplicates warn); keep Underline disabled (Markdown cannot preserve it) and Code disabled (replaced by `extensions/ReviewableCode.ts` so inline code can be tracked as a formatting suggestion). Keep all `@tiptap/*` deps on the same minor version (skew breaks `vite build`); the `overrides` block in `package.json` pins `prosemirror-*` to single instances (a split throws "Adding different instances of a keyed plugin").
- **`hooks/useFileManager.ts`** — All Tauri **file** I/O. Opens/saves `.md` files and manages the parallel `.comments.json` sidecar through the atomic trio. Invoke the file and dialog commands only from here (Claude-streaming and workspace-persistence commands have their own hooks). Every save returns a typed `SaveOutcome` (`saved | blocked{reason:'sidecar-protected'|'baseline-unknown'} | conflict{which, actual} | cancelled | failed{message, path?}`) — never a bare throw. It tracks the on-disk baselines (`expectedDoc` / `expectedSidecar` fingerprints) and detects an **external conflict** by writing with `expected: match(baseline)`; a two-axis save model separates the write precondition (a Save As is unconditional `any`) from fingerprint ownership (only a current-target save advances the baseline immediately; a different Save As target stages then commits only on full success). A **failed save no longer pops a modal** — it returns the typed outcome and the caller presents it source-aware (a manual save shows a dialog; autosave stays quiet). `getIsDirty()` is a synchronous mirror of the dirty state (authoritative right after an awaited save, so the shell's quit guard doesn't race React).
- **`hooks/useSaveCoordinator.ts`** — Per-tab single-flight save coordinator. Every save route (manual, Save As, autosave, conflict resolution) funnels through here so writes are **serialized** (never overlapping), **coalesced** (a burst → one write), and **revision-covered** (a request resolves only once its edit is on disk). `requestSave` (default lane) and `runExclusive` (Save As, which changes the path) share a drain loop; a change arriving during a write schedules exactly one **fresh pass**. `saveAndDrain` (the autosave seam) requests a save, waits for the loop to fully quiesce, and reports the coordinator's **terminal** outcome + the covered revision watermark — so a fresh-pass failure is never masked by an earlier `saved`.
- **`hooks/useAutosave.ts`** — Debounced autosave scheduler (owns only _when_ to save; the coordinator owns the save). 2 s idle debounce with a 15 s max-wait ceiling; single-flight `drive` loop over `saveAndDrain`. Eligibility is a saved path, not mid-conflict, and **dirty** (so a flush on a clean doc is a true no-op). `flush()` joins an in-flight save (used on blur / tab-deactivate / close / quit); a `resetKey` (a scheduler generation bumped on every non-autosave reconciliation) cancels stale timers and drops a late completion; a `stopped` state is latched **by reason** (`conflict` defers to the banner, `blocked` waits for an identity change); a transient `failed` backs off 5 s → 15 s → 60 s and edits during backoff don't reset the debounce; `cancelled` is not a failure. Wired in `DocumentTab` via the single `markDirty` choke point; the footer shows its status.
- **`hooks/useComments.ts`** / **`hooks/useSuggestions.ts`** — CRUD state for comments and track-changes suggestions.
- **`hooks/useDraftAutosave.ts`** — Crash recovery + browser-style session restore (exports `useWorkspaceAutosave`). The shell snapshots the whole workspace — a `WorkspaceFile` of `WorkspaceTab`s, each either a clean saved tab (path only) or a dirty/Untitled tab with an embedded `DraftFile` (content + annotations + chat) — to `workspace.json` in the app data dir: immediately on any change and, while any tab is dirty, every ~5 s (atomic temp-file + rename). Clean sessions are deliberately left on disk so relaunch restores tab order + the active tab; a dirty workspace triggers App's Recover/Discard modal. A legacy single-document `draft.json` is migrated into a one-tab workspace on first read; an unreadable file is quarantined (`quarantine_draft`) rather than overwritten. Outside Tauri every operation is a silent no-op.
- **`hooks/useUpdateCheck.ts`** — Once on launch (production builds only — gated on `import.meta.env.PROD`, so dev/e2e never hit the network), fetches GitHub's latest published release and compares it to `__APP_VERSION__` (injected from `package.json` by `vite.config.ts` **and** `vitest.config.ts` — keep both `define`s). A newer version renders `components/UpdateBanner.tsx` (dismissible; dismissal persisted per-version in localStorage; "View release" opens the browser via the opener plugin). This is notify-only, not an auto-updater. The CSP's `connect-src` allowlists `https://api.github.com` for this.
- **`components/CommentLayer.tsx`** — Two intentionally separate panel strategies. Open uses absolutely positioned, collision-nudged cards translated with document scroll; it owns unresolved comments, the composer, and pending suggestions. All renders every comment in document order as a normal-flow list with its own scrollbar: no anchor catalog, viewport culling, pills, translation, suggestion cards, or document bottom-spacer contribution. Its existing “View suggestion” action switches to Open and defers the jump until that layout mounts. History-card clicks and Unresolve share `utils/commentAnchors.ts`: trust the stored range only while its exact quote remains there, otherwise accept one unique document-wide occurrence, otherwise do nothing (Unresolve shows a transient inline card notice).
- **Fixed document typography + `utils/zoomPreference.ts`** — document body text and headings use bundled Source Serif 4 through `--font-doc-body` / `--font-doc-heading`; Instrument Sans is the UI face and JetBrains Mono is reserved for status/model/session/kbd metadata. `--text-doc-base` fixes the 100% body size at 18px, with the handoff scale expressed in em so the zoom wrapper remains the single multiplier. The faces are imported in `main.tsx` with true document italics and no font CDN — never reintroduce one. There is deliberately no font/size picker: zoom (60–240%) is the only reader-controlled scale, reflows the fixed-width page, and persists globally under `quill-zoom`; the loader also removes the retired `quill-doc-font` / `quill-doc-font-size` keys. UI chrome stays on its own size tokens, print resets screen zoom, and `src/test/styles/documentFonts.test.ts` guards the fixed offline wiring.

### Tiptap Extensions (`src/extensions/`)

- **`TrackChanges.ts`** — ProseMirror plugin that intercepts document transactions in suggesting mode and wraps changes with `tracked_insert` / `tracked_delete` marks. A step that both deletes and inserts (typing over a selection, applied quill-edits) mints both halves with a shared `pairId`: `CommentLayer.tsx` groups them into one `ReplacementCard`, and `acceptChange` / `rejectChange` also accept a pairId, resolving both halves in one transaction. Formatting gestures (bold/italic/strike/**code** — the trackable inline marks registered in `extensions/trackChangesPolicy.ts`; link changes are instead **blocked** in Suggesting mode with a switch-to-Editing notice) are intercepted too: the format applies immediately and a `tracked_format` mark records the per-span net delta `{adds, removes}` so accept = drop markers, reject = invert each span's delta (rejectAll inverts formatting before deleting rejected insertions). One logical format change = one changeId over multiple homogeneous spans (segments split at text-node boundaries; a gesture bridging same-author ids unions them to the oldest; another author's pending format suggestion blocks only the overlapped spans; your own pending insertion takes format raw). `getTrackedChanges` returns the discriminated union `TrackedTextChange | TrackedFormatChange` (format entries carry `segments[]`). The storage also carries an `originCommentId` (set via `setTrackChangesOrigin`, toggled by App's `applyTrackedEdits` like the author): freshly minted changes are stamped with it, so an @claude edit records which comment caused it — persisted as `Suggestion.originCommentId` in the sidecar, surfaced on the cards as a "↳ comment" chip plus a reverse outline while that comment is active, degrading to nothing when the comment is deleted.
- **`Comment.ts`** — Tiptap mark extension for anchoring comment highlights to text ranges.
- **`PendingComment.ts`** — Decoration (not a mark — never touches the document) that keeps the to-be-commented range highlighted while the comment composer is open. Driven by `setPendingCommentRange` / `clearPendingCommentRange`, wired to the composer lifecycle via `AddCommentButton`'s `onComposingChange`.
- **`Find.ts`** — Decoration-only find & replace state (never touches the document); `components/FindBar.tsx` is the UI (Cmd+F). Plugin state `{query, matches, activeIndex}` recomputes matches on every doc change. `findMatches` gathers text per textblock with a char→position map so matches span mark boundaries, but excludes text carrying `tracked_delete` (replace would re-find its own struck-out leftovers) and breaks adjacency at non-text inline nodes. Replacement is ordinary `insertContent`, so TrackChanges mints a tracked replacement pair in suggesting mode; the insert always lands at `match.from` (the struck original follows it), so stepping past a replacement is `match.from + replaceText.length` in both modes. Replace All applies back-to-front in one chain — one transaction, one undo step.
- **`AnnotationFocus.ts`** — Decoration (never touches the document) that intensifies the in-text highlight of the active annotation (comment or suggestion). Driven by `setAnnotationFocus` / `clearAnnotationFocus`, mirrored from App's `activeAnnotation` state; also exports `findAnnotationRange` for live mark ranges. A suggestion target id may be a change id or a replacement's `pairId` (matches both halves). Clicks on annotated text are hit-tested in `Editor.tsx`'s `handleClick` (DOM-walk for `data-comment-id` / `data-change-id`) and reported to App, which focuses the innermost candidate — promoting a replacement half to its pairId.
- **`MarkdownImage.ts`** — Image extension whose `renderHTML` resolves the displayed src at draw time: relative paths are resolved against the open document's directory (`setImageBaseDir`, called by App **before** `setContent` — ProseMirror draws synchronously) through Tauri's asset protocol (`convertFileSrc`). The document attribute keeps the original Markdown path, so serialization is untouched. Outside Tauri or for unsaved docs, relative srcs pass through unresolved. The asset protocol is enabled in `tauri.conf.json` (scope `$HOME/**`, `protocol-asset` cargo feature); the CSP's `img-src` also allows `https:` for remote images. Related: `utils/markdownFidelity.ts` (`detectLossyConstructs`) warns once on open when a file contains footnotes or raw HTML, which the editor would mangle on save.

### Persistence Model

Every saved file produces two files:

- `<name>.md` — Markdown content from Tiptap
- `<name>.comments.json` — Sidecar with comments, suggestions, the document chat thread, the linked AI session, and the linked reference folder (type: `SidecarFile`, version 2)

The sidecar is deleted automatically on save if it contains no data.

Both files are written through the **conflict-aware atomic trio** (`src/utils/atomicFile.ts`): a save writes with `expected: match(baseline-fingerprint)`, so a file that changed on disk since it was opened returns `conflict` instead of silently overwriting. Every save route funnels through the per-tab **`useSaveCoordinator`** (serialized, coalesced, revision-covered), and a saved-path document is **autosaved** by **`useAutosave`** (debounced + flushed on focus-leave/quit). A detected conflict raises the persistent **`ConflictBanner`** (`components/ConflictBanner.tsx`, Overwrite / Save a Copy / Reload); a `blocked` or `failed` save is presented **only for a manual save** (autosave stays quiet in the footer). See the `useFileManager` / `useSaveCoordinator` / `useAutosave` notes above.

Separately, the app-data directory holds `workspace.json` (the multi-tab
session / crash-recovery envelope — see `hooks/useDraftAutosave.ts`), the rotating
`quill.log`, the window-state file, and `session-documents.json`, an atomic
session-id → most-recent saved-document index used only to label the Claude
session picker. `DocumentTab` records a binding whenever both a session and a
real file path exist; missing or malformed indexes never prevent session listing.

### Tauri Backend (`src-tauri/src/lib.rs`)

The Rust layer (`run()`) registers every IPC command — these are the only Rust entry points, so all file system, dialog, and Claude-process work must go through them:

- **File & dialog:** `read_file`, `write_file`, `delete_file` (legacy, still registered), the **conflict-aware atomic trio** the save/open path now uses — `read_file_with_fingerprint` (`{state:'absent'} | {state:'present', content, hash}`; only a missing file is `absent`, every other error rejects), `write_file_atomic({path, content, expected})` → `{status:'written', hash} | {status:'conflict', actual}` (temp-file + fsync + rename; a byte-identical write is a race-checked no-op that skips the rename and preserves the inode; `expected` is `{mode:'any'|'absent'|'match', hash?}`), and `delete_file_if_match({path, expected})` → `deleted | absent | conflict` — plus `show_open_dialog`, `show_save_dialog`, `show_folder_dialog`. The hashing contract is lowercase SHA-256 hex; `src/utils/atomicFile.ts` is the only frontend caller.
- **Workspace / draft persistence:** `write_draft`, `read_draft`, `delete_draft`, `quarantine_draft` manage the `workspace.json` recovery envelope in the app data dir (atomic temp-file + rename; a missing file reads as `None`; `read_draft` falls back to a legacy `draft.json`; `quarantine_draft` renames an unparseable envelope aside). The command names are legacy — the payload is now the multi-tab `WorkspaceFile`, not a single draft.
- **Reference folder:** `list_context_files` walks a linked folder (bounded, hidden/dependency dirs skipped, document-like extensions only, capped at 200) to build the prompt manifest.
- **Claude session integration:** `record_session_document`, `list_claude_sessions`, `read_claude_session_preview`, `find_session_for_markdown`, `check_session_compacted`, `spawn_claude_resume`, `cancel_claude_resume`. The first command atomically updates `session-documents.json`; listing joins that local index so rows lead with the most-recent saved document name, then Claude's optional title, then `untitled-<short-id>`. The remaining commands read `~/.claude/projects/*.jsonl` to locate and preview sessions, and spawn the `claude` CLI (`--resume … --print --output-format stream-json`, optional validated `--model` / `--effort` selections, plus `--add-dir` when a reference folder is linked, with `--` terminating options before the prompt) to stream `@claude` replies back over an IPC `Channel`. Null model/effort values mean Default and omit their flags entirely. `spawn_claude_resume` accepts `allow_create`: when set and the session's jsonl doesn't exist yet, it swaps `--resume` for `--session-id`, creating the session on first contact — this backs the picker's "Start new session" button (`createdByQuill` bindings, which skip the compaction check and always send the full document, with prompts that never claim Claude authored the doc). Spawned children are tracked in a `ChildRegistry` so they can be cancelled.
- **Deep links:** `handle_deep_link` and `take_pending_deep_link` are the IPC commands; both use the internal `parse_quill_open` helper to validate a `quill://open?file=…` URL (a cold-start link is buffered until the frontend subscribes).
- **App lifecycle:** `has_native_menu`, `update_recent_menu`, `exit_app`. The menu's Quit item emits `menu-quit` instead of quitting so the frontend's unsaved-changes guard (`guardDirty` in `App.tsx`, with the `AppModal` Save / Don't Save / Cancel dialog) runs first; the same guard now covers only Quit and window close (New / Open / Open Recent add a tab rather than replacing the open document, so they no longer prompt). Use `AppModal` for any user-facing dialog — `window.alert`/`confirm` are unreliable in Tauri webviews. **Open Recent** is owned by the frontend (`src/utils/recentFiles.ts`, localStorage, capped at 10): `update_recent_menu` rebuilds the whole native menu with the given paths (item ids `recent:<path>` → emitted as `menu-open-recent` with the path payload; `menu-clear-recent` is Clear Menu). The menu event handler is registered once in `setup` — never inside `build_menu`, which now runs per rebuild. Window size/position persist via `tauri-plugin-window-state` (registered in the builder; no JS API used, so no capability entry).
- **Diagnostics & logging:** `get_diagnostics` (app version, OS, arch, log-dir path) and `reveal_logs` (opens the log dir in the OS file manager via the opener plugin). Logs are written by `tauri-plugin-log` (registered in the builder, `log:default` capability): targets `LogDir` (`quill.log` in the app log dir), `Stdout`, and `Webview`, level `Info`, 5 MB cap, `RotationStrategy::KeepOne` (do **not** use `KeepAll` — broken on macOS, plugins-workspace #1397). Emit logs from Rust with the `log` crate macros; from the frontend with `@tauri-plugin-log-api`. `install_panic_hook()` (called at the top of `run()`) routes Rust panics through `log::error!` before the default hook so crashes land in the log file. The **Help** menu exposes "Copy Diagnostics" (`menu-copy-diagnostics` → frontend `handleCopyDiagnostics`, invokes `get_diagnostics` and copies a text block to the clipboard) and "Show Logs" (`menu-reveal-logs` → `handleRevealLogs`, invokes `reveal_logs`); both are wired in App's menu effect alongside the other `menu-*` listeners.

### `@claude` reply flow (`src/hooks/useClaudeReply.ts` + backend)

`useClaudeReply` builds a document-scale prompt — the anchor text as framing ("USER IS COMMENTING ON", not an edit fence), the comment thread, a pending-suggestions listing (App's `getPendingSuggestions`, so Claude doesn't re-propose or conflict with what's already awaiting review), the reference-folder manifest when one is linked, and **always the full current document** (`check_session_compacted` only picks the note wording; skipped for `createdByQuill` bindings) — then calls `spawn_claude_resume` (passing `addDir` for the reference folder, `allowCreate` for Quill-minted sessions, and the global model/effort choices read at spawn time), and consumes the streamed `ChunkEvent`s (`Model` / `Delta` / `Done` / `Error` / `Cancelled`) to update the AI reply in place. `Model` is authoritative stream truth: it stamps that reply and updates the footer's reported-model label independently of the alias selected for future runs. Edits apply with scope `'doc'` and the comment's id as origin. A `quill-edits` entry is a union: `{find, replace}` (text) XOR `{find, format: {bold?, italic?, strikethrough?}}` (formatting; `true` sets, `false` unsets, `strikethrough` maps to the `strike` mark; applied via `setMark`/`unsetMark` in one chain so the TrackChanges engine mints one origin-stamped format suggestion per edit). `planEdits` skips unlocatable finds, text-identical edits, malformed entries (both/neither of replace and format, empty-find format ops), format ops overlapping a text replacement in the same block, and format ops touching another author's pending format suggestion. Both `@claude` comment replies and document chat run over the shared `hooks/useClaudeResumeStream.ts` transport — fence holdback for the `quill-edits` block during streaming lives there; final block extraction is `splitVisible` in `useClaudeReply.ts`.

**Document chat** (`src/hooks/useDocumentChat.ts` + `components/ChatPanel.tsx`): the review panel's **Comments / Chat** header toggle (`components/PanelHeader.tsx`), **Cmd+/**, or the empty-document "press ⌘/ to ask Claude" hint open a whole-document conversation with the linked session. This **replaced** the earlier full-document review dialog — `ReviewModal` / `useDocumentReview` / the `quill-comments` block are gone (a vestigial `is_quill_request` in Rust still matches the old "You are reviewing…" string, but nothing emits it). `buildChatPrompt` always sends the full current document plus the live selection/cursor context, the pending-suggestions listing, and the reference-folder manifest; the composer sends on Cmd+Enter. Chat is **suggestions-only**: it parses just the `quill-edits` block (never `quill-comments`, asserted by a guard test) and applies doc-scoped tracked suggestions via `applyTrackedEdits`, each stamped with `originChatMessageId` — surfaced as a "↳ from chat" chip on the suggestion card (jumps to the message) and a "→ N suggestions" chip on the chat message (jumps to the suggestions). A streaming turn can be Stopped, then Retried or Dismissed. The thread persists in both the sidecar (`SidecarFile.chat`) and the workspace snapshot, keyed to the session id. Chat and `@claude` comment replies share `hooks/useDocumentAIGate.ts` (one concurrent Claude request per document) and the `useClaudeResumeStream` transport.

### Core Types (`src/types/index.ts`)

`Comment`, `Reply`, `Suggestion` (the canonical runtime model is the segment-based `LogicalSuggestion` with `type: 'change'` — there is **no `replacement` type**; legacy `insertion` / `deletion` / `format` variants are accepted read-only at the sidecar boundary), `AISessionBinding`, `ChatMessage` / `DocumentChatThread`, `SidecarFile` (`version: 2`, now with an optional `chat` thread), and the `DraftFile` / `WorkspaceTab` / `WorkspaceFile` recovery envelope — the shared data contract between frontend state and the serialized sidecar. Change these carefully; they affect both runtime state and on-disk files. See `PRD.md` for the full as-built behavior.
